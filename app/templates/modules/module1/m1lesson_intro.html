Introduction of Code Comments

What are Code Comments?
Code comments are notes you write in your code to explain what your code does, why you made certain choices, or to leave reminders for yourself or others. Comments are ignored by the computer when your program runs, but they are essential for making your code readable and maintainable.

Why Comments Matter?
Comments help you and others understand your code, especially when you come back to it after some time. They make it easier to spot mistakes, explain tricky logic, and help teams work together on larger projects.
Python

# This function adds two numbers

Why Comments Matter
  • Clarity: Comments help others (and your future self) understand complex code.
  • Maintainability: Good comments make it easier to update and debug code.
  • Collaboration: Comments communicate intent and important details to teammates.
The Comment Hierarchy
  1. Best: Self-documenting code that needs no comments
  2. Good: Clear code with minimal, meaningful comments
  3. Acceptable: Comments to explain complex logic or intent
  4. Bad: Excessive comments explaining obvious code
  5. Worst: Misleading or outdated comments
When to Comment
  • To explain why something is done, not just what the code does
  • For complex algorithms, business rules, or workarounds
  • To warn about side effects or important caveats
  • For documenting APIs, parameters, and return values
Best Practices for Comments
  • Be concise and clear. Avoid unnecessary words and get to the point.
  • Keep comments up to date. Revise or remove comments as code changes.
  • Use proper grammar and spelling. Professional comments are easier to read and maintain.
  • Use TODO, FIXME, NOTE tags to highlight areas needing attention.
  • Remove commented-out code. Use version control for history.
  • Use the correct syntax: # for Python, // or /* ... */ for JS, <!-- ... --> for HTML.
Comment Maintenance
  • Review comments during code reviews
  • Delete or update comments for removed or changed code
  • Strive for code that needs fewer comments by improving naming and structure
