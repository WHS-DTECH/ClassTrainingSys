Introduction to Debugging

What is Code Debugging?
Code debugging is the process of identifying, isolating, and fixing errors or bugs in your code. It involves understanding how your program works, finding where things go wrong, and correcting mistakes to ensure your code runs as intended.

Why does Debugging Matter?
Debugging is a critical skill for programmers because all codeâ€”no matter how experienced the developerâ€”will have bugs at some point. Effective debugging helps you solve problems faster, learn from mistakes, and produce reliable, maintainable software. It also builds your problem-solving skills and persistence, which are essential for success in programming.
Golden Rule
The Golden Rule of Debugging
Donâ€™t make more than one change at a time.
When debugging, always change only one thing in your code or environment before testing again. If you change multiple things at once, you wonâ€™t know which change fixed (or broke) your code. This rule helps you isolate the cause of a bug and avoid introducing new problems.

  • Always reproduce the bug before trying to fix it.
  • Understand the problem fully before making changes.
  • Test after each change.
Debug Examples
Bad Debug Example
# Bad: Making multiple changes at once and not checking results
def add(a, b):
    return a - b  # Bug: should be a + b
# Developer changes both the function and unrelated code, then tests.
Good Debug Example
# Good: Change one thing, test, and document
def add(a, b):
    return a + b  # Fixed: changed '-' to '+'
# Test this change before making any others.
When should I debug?
When to Debug
  • When your code produces errors or unexpected results.
  • When a feature stops working after a change.
  • When you want to understand how a piece of code works.
  • When you need to optimize or refactor code.

Practice: Where Would You Debug?
  1. Example 1:
    def greet(name):
    print("Hello" + name)

greet()

    Where would you debug? (describe the line or issue):
    You should debug the function call greet() because it is missing an argument. The error will be a TypeError.
  2. Example 2:
    numbers = [1, 2, 3, 4, 5]
for i in range(6):
    print(numbers[i])

    Where would you debug? (describe the line or issue):
    You should debug the line print(numbers[i]) because when i=5, numbers[5] is out of range. The error will be an IndexError.
  3. Example 3:
    def divide(a, b):
    return a / b

print(divide(10, 0))

    Where would you debug? (describe the line or issue):
    You should debug the call print(divide(10, 0)) because dividing by zero will cause a ZeroDivisionError.
  4. Example 4:
    total = 0

    total = total + i
