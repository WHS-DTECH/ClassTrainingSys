üí¨ Understanding Comments in Code

Why are comments important?
Comments are essential for making code readable, maintainable, and understandable by others (and your future self). They explain the purpose, structure, and logic of your code, making it easier to debug, extend, and collaborate on projects. Good comments help teachers, peers, and even automated tools understand what your code is doing and why.

Types of Comments
There are several types of comments, each serving a different purpose. The four feedback boxes on the right show examples for Python, Javascript, HTML, and CSS:
  • File or Program Header Comments: At the top of a file, these describe the program's name, purpose, inputs, and outputs. (See the top section of each feedback box.)
  • Section or Function Comments: Placed above a section of code or a function, these explain what that part does. (See the 'Section Comment' or 'Function Comment' in each box.)
  • Inline Comments: Short notes on the same line as code, clarifying a specific statement. (Look for lines with #, //, or /* ... */ at the end of a code line.)
  • DEBUG Comments: Used during testing to record what was tested, any issues found, and how they were fixed. (See the 'DEBUG Comment' in each box for real debugging notes.)
  • Accessibility/UX Comments (HTML only): Special comments to explain user interface or accessibility features, helping others understand how the UI supports users. (See the HTML box for an example.)
Tip: Use comments generously but meaningfully. Avoid obvious comments (e.g., # increment x for x += 1), but always explain complex logic, important decisions, and the overall structure of your code.
