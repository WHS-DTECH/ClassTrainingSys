Assessment Checking

Using a Debug Comment Block

Debug comment blocks help you organize your debugging process and communicate your thought process clearly. Use the following layout in your code: 

# DEBUG TEST: Describe the test you are running (e.g., input values, expected output)
# DEBUG ISSUE: Clearly state the problem or bug you found
# DEBUG FIX: Explain the change you made to fix the issue
  • # DEBUG TEST: What did you test? (e.g., "Tested with input x=5, expected result 25.")
  • # DEBUG ISSUE: What was the problem? (e.g., "Output was 20 instead of 25 due to missing multiplication.")
  • # DEBUG FIX: How did you fix it? (e.g., "Corrected the formula to x*x instead of x+15.")

This structure makes your debugging process transparent and helps teachers and peers understand your approach.

What is Assessment Checking?

Assessment checking is the process of reviewing student work to ensure it meets the required criteria and learning objectives. It involves:

  • Verifying code correctness and functionality
  • Checking for adherence to style and conventions
  • Ensuring documentation and comments are present
  • Providing constructive feedback for improvement

Best Practices for Checking

  • Use a checklist or rubric for consistency
  • Test code with a variety of inputs
  • Look for edge cases and error handling
  • Encourage students to self-check before submission
  • Give specific, actionable feedback

Common Mistakes to Watch For

  • Missing or incorrect output
  • Poor variable/function naming
  • Lack of comments or documentation
  • Hardcoded values instead of variables
  • Not following assignment instructions

Tips for Students

  • Read the assessment criteria carefully
  • Test your code before submitting
  • Ask for feedback and revise your work
  • Check your comments and documentation
  • Reflect on what you learned from the task
