Assessment Checking Overview

Types of Documentation & Comments

  • Inline Comments: Short notes placed directly in code to clarify a specific line or logic. Example: # increment counter
  • Header Comments: Placed at the top of a file or function, describing its purpose, author, date, and important details. Example: """This script processes student grades. Author: Ms. Smith, 2025"""
  • README / Program Overview: A separate file or section that explains what the program does, how to use it, and any setup instructions. Example: README.md in a project folder.
  • Test Logs: Notes or files that record the results of running tests, including what was tested, expected vs. actual results, and any issues found. Example: test_log.txt or comments like # Test: input 0, output OK
Tip: Good documentation helps others (and your future self) understand, use, and maintain your code!
Is the comment clear and descriptive?
Clarity and Descriptiveness
Is the comment easy to understand and does it clearly describe the code’s intent?
Avoid ambiguous or cryptic comments.
Does it explain why, not just what?
Explains “Why”, Not Just “What”
Good comments explain the reasoning behind code decisions, not just what the code does.
Example: Instead of “# increment x”, use “# increment x to avoid off-by-one error”.
Is it meaningful, specific, and helpful?
Meaningful, Specific, and Helpful
Comments should add value and context, not restate the obvious.
Example: “# Fetch user data from API for caching” is better than “# get data”.
Is it too vague or redundant?
Avoids Vagueness and Redundancy
Don’t use comments like “# loop” or “# variable” that add no new information.
Avoid repeating what the code already says.

Practice: Compare Documented vs Undocumented Code

Documented Program

# Calculate the average of a list of numbers
def average(numbers):
    """Return the average of a list of numbers."""
    if not numbers:
        return 0
    total = sum(numbers)
    return total / len(numbers)

data = [10, 20, 30, 40]
print("Average:", average(data))

Undocumented Program

def a(x):
    if not x:
        return 0
    t = sum(x)
    return t / len(x)

z = [10, 20, 30, 40]
print(a(z))
